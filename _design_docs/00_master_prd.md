## 1. 프로덕트 비전 & 원칙 (Vision & Principles)

### 1.1 비전
"기타를 배우고 싶은데 돈이 없어서 못 배우는 사람이 지구상에 한 명도 없게 만든다."

유튜브 강의는 산발적이고, 유료 레슨은 비싸다. 우리는 **체계적인 커리큘럼(월→주→일 단위)** 을 **유료 레슨급 품질**로, **완전 무료**로 제공한다. 수익은 오직 자발적 도네이션에서만 발생하며, 그마저도 사용자에게 부담을 주지 않는 방식으로 유도한다.

### 1.2 5대 설계 원칙 (Non-Negotiables)
1. **비용 0원 원칙 (Zero-Cost):** 서버, DB, 인증 서버를 두지 않는다. 순수 정적 사이트 + 브라우저 `localStorage`만으로 동작한다.
2. **토스(TOSS) 톤앤매너 원칙:** 모든 UI와 문구는 극도의 미니멀리즘, 넓은 여백, 친절한 대화체를 따른다. 매뉴얼조 금지.
3. **할루시네이션 제로 원칙 (Zero-Hallucination):** 지판·악보는 이미지 생성 AI로 절대 만들지 않는다. 정형화된 JSON 데이터 → 프론트엔드 렌더링으로만 그린다.
4. **모듈형 확장 원칙 (Modular):** 커리큘럼은 콘텐츠 데이터일 뿐이다. 새 커리큘럼 추가가 '코드 수정'이 아니라 '데이터 추가'로 끝나야 한다.
5. **글로벌 3개국어 원칙 (i18n-first):** KR/EN/JP를 1급 시민으로 취급한다. 문자열 하드코딩 금지.

### 1.3 설계-구현 분리 원칙
- `_design_docs/` : 기획·설계·콘텐츠 원본(SSOT, Single Source of Truth). CC(에이전트)의 주 작업 공간.
- `web_app/` : 실제 프론트엔드 구현체. `_design_docs`가 확정된 뒤에야 코드를 생성한다.
- 콘텐츠(레슨 텍스트·악보 JSON)는 `_design_docs`에서 작성/검수 → 빌드 시 `web_app`으로 이식되는 파이프라인을 따른다.

## 2. 타깃 사용자 & 페르소나 (Target Users)

| 페르소나 | 상황 | 핵심 니즈 | 우리가 주는 것 |
|---|---|---|---|
| **입문자 지훈 (25)** | 기타를 사고 싶은데 독학이 막막함 | "뭐부터 어떻게?" | 손 잡아주는 일 단위 커리큘럼 |
| **중급 정체기 사라 (31, EN)** | 코드는 치는데 솔로/스케일이 안 늘음 | 체계적인 스케일 훈련 | 3개월 솔로/스케일 로드맵 |
| **재입문 유키 (40, JP)** | 예전에 치다 접음, 다시 시작 | 부담 없는 재시작 | 위트 있는 격려 + 무료 |

**공통점:** ① 돈 쓰기 싫음 ② 꾸준함이 어려움(이탈 방지 UX가 핵심) ③ 모바일 우선 사용.

## 3. 핵심 기능 명세 (Feature Specification)

### 3.1 메인 랜딩 페이지
- 첫 화면에서 **"모든 콘텐츠는 100% 무료입니다"** 를 가장 크고 명확하게 선언.
- 커리큘럼 카드들이 세로로 나열(모바일 우선). 현재는 `3개월 솔로/스케일` 1개, 하단에 "곧 추가될 커리큘럼"(잽잽이/펑키 리듬, 스트로크 주법) 자리를 비워둔 티저 카드.
- 상단 우측: 언어 스위처(KR/EN/JP), 닉네임(설정 시 표시).
- 하단: 도네이션 CTA(과하지 않게), 프로젝트 철학 한 줄.

### 3.2 커리큘럼 계층 구조 (Curriculum Hierarchy)
```
카테고리(Curriculum) → 월(Month) → 주(Week) → 일(Day, 레슨 상세)
```
- 각 계층은 `overview`(개요) 페이지를 가진다.
- **Day = 최소 학습 단위**이자 진도 체크의 원자(atomic) 단위.

### 3.3 레슨 상세 페이지(Day) 구조 — 고정 템플릿
1. **① 이론/설명** — 토스체로 "왜 이걸 배우는지" 먼저. 개념 → 손 모양 → 주의점.
2. **② 시각 자료** — 지판 다이어그램(Fretboard) 및/또는 타브 악보(Tab). **JSON 데이터로부터 렌더링** (3.5 참조).
3. **③ [오늘의 연습 완료] 액션 버튼** — 클릭 시:
   - 팡파르(컨페티) 애니메이션 재생.
   - `localStorage` 진도율 갱신.
   - 조건 충족 시 유동 알림 팝업(3.4) 트리거.
   - 다음 Day로의 자연스러운 이동 유도.

### 3.4 진도 관리 & 유동 알림 (Progress & Micro-Nudges)
- 진도 데이터는 `localStorage`에 JSON으로 저장(스키마: `01_architecture/state_storage.md`).
- **트리거 기반 마이크로카피**(이탈 방지 핵심 장치):
  - 예) 2주차 진입 시: *"약 80%의 사람들은 1주차에서 포기했어요! 여기까지 온 이상 당신은 상위 20%에 진입한 거예요. 다음 내용도 같이 달려봐요!"*
  - 트리거 조건과 문구는 데이터로 관리(`04_localization/translation_map.md`) → 코드 수정 없이 추가/수정 가능.
- **아낀 레슨비 카운터:** 완료한 Day 수 × 가상 단가로 "지금까지 약 XX만 원을 아꼈어요"를 계산해 도네이션 유도에 활용.

### 3.5 할루시네이션 없는 시각 자료 렌더링 (핵심 기술 차별점)
- **금지:** Midjourney/DALL·E 등 이미지 생성 AI로 지판·악보 생성 (줄 개수/프렛 위치 오류 발생).
- **채택:** *데이터 기반 렌더링(JSON → UI)*.
  - 외부 전문 AI(GPT-4o 등)로 **음 좌표·프렛·스트링 번호**를 표준 JSON 스키마로 추출(`03_data_schema/fretboard_score_schema.json`).
  - 프론트엔드가 이 JSON을 받아 **SVG(지판 다이어그램)** 및 **VexFlow / AlphaTab(타브 악보)** 로 정확히 렌더링.
- 이 방식의 이점: 픽셀이 아니라 데이터라서 **검증 가능**하고, 다국어/다크모드/반응형에 자동 대응된다.
> ✅ **구현 확정:** 지판·타브 **둘 다 자체 SVG 렌더러**로 구현(AlphaTab/VexFlow 미채택). 표준 스키마 = `03_data_schema/fretboard_score_schema.json`, 역할→색 = `03_data_schema/color_legend.md`. 향후 재생 필요 시 자체 Web Audio, 오선보 필요 시 VexFlow.

### 3.6 다국어 지원 (i18n)
- 지원 언어: **한국어(ko) / 영어(en) / 일본어(ja)**.
- 사전 구조: `ko.json`, `en.json`, `ja.json` (중첩 키 딕셔너리).
- UI 문자열·마이크로카피·도네이션 문구 전부 사전 경유. 하드코딩 금지.
- 언어 선택은 `localStorage`에 저장, 브라우저 언어 자동 감지 후 폴백(ko).

### 3.7 도네이션 시스템
- 노출 위치: 각 주차/커리큘럼 완료 시점(성취감이 최고조인 순간)에 위트 있게.
- 문구 예: *"지금까지 약 XX만 원의 레슨비를 아꼈어요! 앞으로도 전부 무료예요. 다만 개발자에게 커피 한 잔(☕) 정도의 후원은 큰 힘이 됩니다."*
- 채널: **Toss 송금 링크 / 카카오페이 QR·송금 / PayPal / Buy Me a Coffee**.
  - 국내(ko): Toss·카카오페이 우선 노출. 글로벌(en/ja): PayPal·BMC 우선 노출.
- 강매·모달 강제 없음. 언제나 "닫고 계속 무료 이용" 경로 제공.
> ✅ **구현 확정:** 실제 채널 = **카카오페이·토스·PayPal** 3종(BMC는 한국 미지원으로 제외). 각 채널 **QR/링크 토글**(기기별 기본값: PC=QR, 모바일=링크). 큰 마일스톤(월·커리큘럼 완료) 시 **'드림기타' 후원 연출** → 채널 시트. 결제 뷰에 세무·실명 고지 캡션(차단형 아님).

## 4. 정보 구조 & 라우팅 (IA & Routing)

정적 사이트 기준 URL 설계(해시 또는 히스토리 라우팅):
```
/                                   랜딩(커리큘럼 목록)
/c/solo_scale_3months               커리큘럼 개요
/c/solo_scale_3months/m1            월 개요
/c/solo_scale_3months/m1/w1         주 개요
/c/solo_scale_3months/m1/w1/d1      레슨 상세(Day)
/about, /donate, /disclaimer        정적 페이지
```
- 콘텐츠 소스는 `_design_docs/02_curriculum/**`의 md + JSON. 빌드 시 라우트로 매핑.
> ✅ **구현 확정(Astro 정적 라우팅):** 레슨마다 **실제 정적 HTML**을 생성(SEO). 실제 URL 스킴 = `/c/{cur}/m{M}/w{W}/d{D}/`. **언어별 라우트**: ko는 무접두(`/…`), 영어 `/en/…`, 일본어 `/ja/…` + canonical·hreflang. 월/주 개요는 별도 페이지가 아니라 커리큘럼 뷰의 **인라인 아코디언**(Month→Week→Day). 총 163 정적 페이지(레슨 156 + 홈/커리큘럼 6 + 404/robots/sitemap).

## 5. 시스템 아키텍처 개요 (상세: `01_architecture/`)

- **프론트엔드:** 정적 SPA 또는 SSG. (후보/결정: `tech_stack.md`)
- **상태/진도:** `localStorage` JSON 단일 소스. (스키마: `state_storage.md`)
- **콘텐츠:** md(설명) + JSON(악보/지판) 파일 기반.
- **배포:** Cloudflare Pages(1순위) / GitHub Pages(대안), 기본 도메인(`*.pages.dev` / `*.github.io`) 사용.
- **비용:** 호스팅·DB·인증 전부 $0.

### 5.1 LocalStorage 한계의 정직한 고지 (필수 UX)
브라우저 캐시 삭제 시 진도 유실, 기기 간 동기화 불가라는 한계를 **숨기지 않고 토스 스타일로 위트 있게** 안내한다.
- 예: *"기록은 이 브라우저에만 저장돼요. 시크릿 모드나 캐시 삭제엔 약해요. (로그인 없는 무료 서비스의 낭만이랄까요 😌) 중요한 진도는 가끔 캡처해두세요!"*
- 옵션: 진도 JSON **내보내기/불러오기(Export/Import)** 버튼으로 유실·이전 위험을 완화.

## 6. 데이터 & 콘텐츠 생산 파이프라인 (AI Content Pipeline)

우리는 콘텐츠를 코드가 아니라 **외부 전문 AI 세션**에서 생산해 로컬로 수합한다.

```
[세션 A: GPT-4o]  악보/지판 JSON + 레슨 콘텐츠  ─┐
[세션 B: Claude ]  토스체 3개국어 마이크로카피   ─┼─▶ _design_docs/ 수합 & 검수 ─▶ web_app/ 빌드 이식
[세션 C: Gemini ]  세무/법률 리스크 리포트       ─┘
```
- 각 세션 초기 프롬프트는 `03_data_schema` / `04_localization` 스키마에 **강제 정합**되도록 설계(부록 A 참조).
- 수합된 산출물은 스키마 유효성 검사를 통과해야 `web_app`으로 승격.

## 7. 수익 & 세무/법률 로드맵 (Monetization & Tax)

원칙: 서비스는 영구 무료. 수익은 도네이션만. 하지만 **소득 발생 = 신고 의무**가 생기므로 단계적으로 관리한다. (상세 리서치: 세션 C 산출물 수합 완료 → `tax_legal_report.md` 참조)

| 단계 | 상황 | 형태 | 세무/법률 액션 |
|---|---|---|---|
| **0. 시드** | 후원 미미 | 개인 후원 | 사업자 등록 X. 기록만 보관 |
| **1. 성장** | 유의미한 소득 발생 | 개인(기타/사업소득) | 종합소득세 신고 준비, 증빙 정리 |
| **2. 확장** | 지속·반복 소득 | 간이과세자 전환 검토 | 사업자 등록, 부가세/소득세 체계화 |
| **3. 글로벌** | 해외 후원(PayPal 등) | 외화 수취 | 외환·원천 이슈, 플랫폼별 과세 검토 |

> ⚠️ 본 문서는 방향성 로드맵이며 세무 자문이 아니다. 실제 신고 전 **세무사 확인 필수**. 근거 자료: [`tax_legal_report.md`](./tax_legal_report.md) — 후원금 성격 분류, 사업자 등록 리스크, 소득 규모별 로드맵, 해외 후원 유의사항, 증여세 이슈, 실무 체크리스트 수록. (본 리포트 역시 방향성 자료이며 실제 신고 전 세무사 확인 필요)

## 8. 비기능 요구사항 (Non-Functional)

- **성능:** 정적 자산 + CDN, 첫 로드 Lighthouse 90+ 목표. 악보는 지연 로드.
- **접근성:** 명도 대비, 키보드 내비게이션, 시맨틱 마크업.
- **반응형:** 모바일 우선(대부분 스마트폰으로 연습하며 봄).
- **오프라인 내성(선택):** 정적 특성상 PWA/캐시로 오프라인 열람 확장 여지.
- **개인정보:** 닉네임·진도는 로컬에만 저장. 서버 전송·수집 없음(프라이버시 by design).

## 10. 리스크 & 대응 (Risks)

| 리스크 | 영향 | 대응 |
|---|---|---|
| localStorage 진도 유실 | 사용자 이탈/불만 | 정직한 고지 + Export/Import |
| AI 악보 오류(할루시네이션) | 신뢰도 치명타 | JSON 스키마 강제 + 검수 + 렌더 검증 |
| 세무 미신고 | 법적 리스크 | 단계별 로드맵 + 세무사 확인 |
| 콘텐츠 생산 병목 | 진척 지연 | 세션 분업 + 스키마 표준화로 병렬 생산 |
| 토스 톤 일관성 붕괴 | 브랜드 훼손 | 카피 전량 사전(dictionary) 경유 + 리뷰 |
