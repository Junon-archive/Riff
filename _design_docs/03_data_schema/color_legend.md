# 지판/타브 렌더러 색 범례 (Role → Color) — SSOT

> 렌더러(`web_app/src/render/fretboard.ts`·`tab.ts`)가 음의 **role**에 따라 칠하는 실제 색.
> **산문(레슨 본문)에서 색을 언급할 때는 반드시 이 표의 색을 쓴다.** (지판에 그려진 색과 글이 일치해야 함)

| role / 표기 | 실제 렌더 색 | 산문에서 쓸 색 단어 (ko / en / ja) |
|---|---|---|
| `root` / `isRoot` (근음) | 파랑 (var--primary) | 파랑 / blue / 青 |
| `chord_tone` (코드톤 1·5 등) | 파랑 (var--primary) | 파랑 / blue / 青 |
| `target` (코드 3도 착지) · `highlight`(일반 강조) | **초록** (var--green-500) | 초록 / green / 緑 |
| `color` (색채음: 도리안 ♮6, 믹솔리디안 색채 등) | 노랑 (var--yellow-500) | 노랑 / yellow / 黄 |
| `blue_note` (블루노트 ♭5·♭3) | 보라 (var--rn-blue-note) | 보라 / purple / 紫 |
| `scale` / `passing` / 미지정 | 채움 없음(외곽선, 현색) | 색으로 부르지 말 것(“표시된 음” 등) |

## 규칙
- 산문이 특정 음을 색으로 지목할 때 그 음의 role을 보고 위 색을 쓴다.
  - 예) "타겟 ♭3(C)" = target → **초록**. "근음 A" = root → **파랑**. "도리안 ♮6(F#)" = color → **노랑**. "블루노트 ♭5" = blue_note → **보라**.
- "빨간/빨강/red/赤" 등 렌더러에 없는 색은 쓰지 않는다.
- 색보다 **역할·도수 라벨**로 부르는 것이 더 안전하다(예: "강조된 ♭3", "타겟 표시된 음"). 색을 쓸 거면 위 표를 정확히 따른다.

> 배경: 초기 세션 A 콘텐츠가 임의 색(주로 "빨강=타겟")을 써서 실제 렌더 색(초록)과 어긋났다. 이 표를 기준으로 산문을 교정한다(2026-07-03 스윕).
