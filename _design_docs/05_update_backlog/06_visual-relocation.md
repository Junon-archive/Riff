# 06 · 기존 커리큘럼 악보 인라인 재배치 (콘텐츠 편집)

## 목적 (왜)

백로그 01 Part A로 이제 악보를 ①이론/③연습 어디서든 저작 위치에 인라인 렌더할 수 있다. 그런데 기존 3커리큘럼은 구 파이프라인(악보=②시각자료 전용) 아래 저작돼, **①이론이 폼/좌표를 설명하며 "아래 다이어그램"을 가리키는데 그 악보는 ②에만** 있는 "설명↔그림 분리"가 광범위하다. 이 문서는 그 악보들을 설명 옆으로 옮기거나(복제) 연습 스텝에 추가하는 **콘텐츠 편집 작업**을 관리한다.

**성격:** 렌더러·스키마·검증기 무변경. `_design_docs/02_curriculum/**/day_*.{ko,en,ja}.md` 의 ```json 블록 **위치만** 옮기거나 복제. 산문 문장은 수정하지 않는다.

## 근거 조사 (SSOT)

전 116 day 전수 검토(서브에이전트 3, 커리큘럼별). 총 **78 제안 day**(상 17 / 중 44 / 하 17), 적정 38. 상세 표는 조사 원본에 있다 — 실제 작업 시 아래 표를 그대로 따른다. (원본은 임시 `/tmp/t9_review/{00_SUMMARY,solo,chord,funk}.md` 에 있었으므로, 착수 시 이 문서의 표가 정본이다. /tmp 는 휘발성이니 아래에 상 등급을 옮겨 적어 둔다. 중·하 등급 전체 목록은 착수 시 재조사하거나 이 문서를 확장한다.)

## 편집 규칙 (확정)

1. **위치만 이동/복제:** ②의 ```json 블록을, 그것을 가리키는 ①이론 문장 **바로 다음(빈 줄 사이)** 또는 ③연습 해당 스텝 다음으로 옮기거나 복제한다. 산문은 안 건드린다.
2. **이동 vs 복제:**
   - **이동** = ②에서 빼서 ①/③로. ②에 그 악보가 더는 필요 없을 때(설명용이 주목적).
   - **복제** = ②에 남기고 ①/③에도 같은 블록. ②가 "예제 모음"으로 여전히 필요할 때.
   - ★**복제해도 안전(검증 완료):** 같은 악보 블록을 한 day에 2번 둬도 scores는 위치 기반 전역 인덱스라 각각 별개 슬롯으로 정상 렌더된다(id 충돌·접미사 불요). 빌드 테스트로 확인함.
3. **3언어 동시:** ko/en/ja 세 파일의 **같은 위치**에 동일 편집. 블록 JSON 은 바이트 동일.
4. **리듬/데이터 불변:** 블록 내용은 그대로. 위치만 바뀐다.
5. **③ 재노출 억제:** ③이 "예제 N 반복"이라 지시하고 그 블록이 ② 바로 위에 인접하면 재인라인 이득이 작으니 하지 않는다. ③ 추가는 "좌표를 글로만 나열하는 두뇌훈련" 자리에 한정.

## 검증 게이트 (각 커밋)

- **V1** `npm run build` exit 0.
- **V2** 편집한 day 의 ko/en/ja ```json 블록 개수·내용 동일.
- **V3** `node web_app/scripts/check-invariants.mjs` 회귀 0 — ★블록을 **이동만** 하면 day 의 scores 집합이 그대로라 지문 불변. **복제**하면 그 day 의 노트헤드/블록 수가 늘어 지문이 바뀌므로, 복제한 day 는 예상된 증가임을 확인하고 `--update` 로 baseline 갱신 후 diff 를 리뷰(다른 day 는 불변이어야 함).
- **V4** 편집 day 렌더 스냅샷 — 악보가 의도한 섹션·위치에 뜨는지, 미치환 슬롯 0.
- **V5** 미편집 day·미대상 커리큘럼 바이트 불변.

## 착수 전략 (권장)

- **파일럿 1 day 먼저:** funk 상 등급 중 하나(예: m1.w4.d1 e9_form — ①이 "아래 다이어그램에서 노랑…근음은 5번 줄 7프렛"으로 명시 지목)로 이동/복제·3언어·V게이트 전 과정을 확립.
- **파일럿에서 ①↔② 실제 세로거리를 렌더로 실측** → 이동 이득이 실제로 큰지 확인 후, 클수록 상→중→하 순 확대. 이득이 작으면 상 등급만 하고 중·하는 보류.
- 커리큘럼별·우선도별로 배치(예: funk 상 8 → chord 상 5 → solo 상 4 → 중 등급…). 각 day 또는 소묶음 1커밋.
