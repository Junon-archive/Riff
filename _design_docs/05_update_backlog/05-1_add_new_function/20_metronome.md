# 20 · 메트로놈 (Metronome) + Functions 공통 셸

## 용어

- **Functions:** 커리큘럼 외 추가되는 연습 도구 묶음(메트로놈·배킹·튜너·루프스테이션…)의 총칭.
- 이 문서는 (A) **Functions 공통 셸**과 (B) **메트로놈 기능**을 함께 정의한다.

## 목적 (왜)

연습 중 **커리큘럼 페이지를 떠난 느낌 없이** 바로 켜고 끄는, 소리가 불쾌하지 않은 메트로놈. Toss 톤(미니멀·깔끔)·Zero-Cost(정적, 서버·DB 없음)·i18n 3언어·기타(파랑)/베이스(초록) 테마를 그대로 지킨다.

### A1. 아키텍처 — "레슨 이동 시 정지"가 설계를 단순화한다

- 사이트는 `<ClientRouter/>`(Astro View Transitions) 소프트 내비. **레슨 이동 시 정지가 요구**이므로, 오디오 엔진을 전환 간 살려둘 필요가 없다 → **`transition:persist`·전역 싱글턴 지속 로직 불필요**.
- **정지 배선:** 오디오 엔진 모듈 싱글턴을 두되, `astro:before-swap`(또는 `astro:before-preparation`)에서 `engine.stop()` + FAB 상태 리셋. "같은 페이지 안 지속"은 DOM/JS 자연 수명으로 자동 충족(오버레이만 닫고 엔진은 계속 발음).
- **지연 로드(정적 구성 유지 핵심):** FAB·오버레이 골격은 가볍게 상주, **오디오 엔진 코드(스케줄러·합성·샘플 디코드)는 FAB 최초 클릭 시 `import()` 지연 로드**. → 메트로놈/배킹 안 쓰는 페이지뷰는 JS·오디오 비용 0. 지금의 가벼운 정적 체감 유지.
- **재사용 컴포넌트:** 메트로놈/배킹 UI는 **(a) 레슨 내 플로팅 오버레이**와 **(b) 전용 페이지** 양쪽에서 같은 코어를 쓴다. 셸(오버레이 vs 풀페이지)만 다르고 내부 패널은 공용.

### A2. FAB 노출 범위 (확정)

- **커리큘럼 세부/레슨 페이지에서만** FAB 노출. **랜딩 제외**(랜딩은 카드 행으로 진입). 전용 도구 페이지에도 FAB 불필요(이미 그 도구 화면).

### B1. 컨트롤 구성 (잠정 레이아웃)

- 큰 **BPM 숫자** + −/＋ 및 슬라이더, **Tap 버튼**.
- **박자 선택**(2/4·3/4·4/4·6/8) 세그먼트.
- **세분 선택**(♩ / ♪♪ / 셋잇단 / 16분) 세그먼트.
- **음색 선택**(일반·드럼·목소리).
- **볼륨 슬라이더**(마스터 게인).
- **시각 박 표시**(원 N개, 강박 파동).
- **재생/정지**.
- 마지막 설정(BPM·볼륨·음색·박자·세분)은 `storage.ts` 경유 저장 → 다음에 그대로.

### B2. 기술 검토 — 타이밍(핵심)

- `setInterval`로 직접 발음 = **박 드리프트/백그라운드 throttle**. **금지.**
- **Web Audio 룩어헤드 스케줄러**(Chris Wilson "A Tale of Two Clocks"): 느슨한 `setInterval(~25ms)`가 미래 ~100ms를 미리 보고 각 박/세분을 `AudioContext.currentTime`(샘플 정확)에 **예약**. 하드웨어 클록이 타이밍 보장 → 드리프트 0.
- **시각 동기:** 예약된 `{beat, subdiv, time, isAccent}` 큐를 `requestAnimationFrame`이 읽어 `currentTime`이 지날 때 글로우/파동 트리거. (오디오와 별도 타이머로 그리면 눈·귀 어긋남.)

### B3. 기술 검토 — 음색(하이브리드)

- **일반 클릭 = 합성(자산 0KB):** `OscillatorNode`+`GainNode` 엔벨로프(빠른 attack·지수 감쇠). 강박=높은 음/큰 게인, 약박=낮은 음. 세분음은 더 여리게. → 예쁜 클릭을 파일 없이.
- **드럼·목소리 = 샘플:** 짧은 mono 파일을 빌드 자산으로 로드→`decodeAudioData`→`AudioBuffer` 캐시. 매 발음마다 **새 `AudioBufferSourceNode`**.
  - **"빠른 템포에서 목소리 겹침/잘림" 해결됨:** 매 박 새 노드라 샘플이 박 간격보다 길어도 **겹쳐 재생(끊김 없음)**. (단일 `<audio>` 방식의 문제였음.) 매우 빠를 때 처리(겹침 허용/하드컷/특정 BPM 이상 비활성)는 아래 Q.
- **샘플 슬롯 계약:** 아래 §B3-1 로 확정(구현 완료).

### B3-1. 샘플 슬롯 계약 (확정 2026-07-24 · 구현 완료 — CC0 확보 세션은 이 규약만 보면 됨)

**둘 곳(고정 경로).** `web_app/public/` 아래에 그대로 떨군다. 빌드 타임에 `src/lib/samples.server.ts` 가 존재를 탐지해 패널의 `data-samples` JSON 으로 내려주고, 엔진이 그 URL 을 `fetch → decodeAudioData → AudioBuffer` 로 캐시한다.

| 슬롯 | 경로 | 쓰임(매핑) |
|---|---|---|
| 킥 | `public/metronome/drum/kick.m4a` | **강박**(모든 박자의 1박) |
| 스네어 | `public/metronome/drum/snare.m4a` | **백비트** — 4/4의 2·4박, 2/4의 2박, **6/8은 4번째 8분**(부강박). 3/4엔 없음 |
| 하이햇 | `public/metronome/drum/hihat.m4a` | 나머지 약박 + **모든 세분음**(게인 0.22로 아주 여리게) |
| 목소리 1~6 | `public/metronome/voice/1.m4a` … `6.m4a` | 펄스 번호 그대로(`1.m4a`=1박). **6개 필요**(6/8의 8분 6펄스가 최대) |

**규칙(중요).**
- **표준 확장자 = `.m4a`(AAC).** 다른 포맷도 자동 인식한다 — 탐색 우선순위 `m4a → mp3 → ogg → wav`. 같은 슬롯에 여러 포맷을 두면 앞선 것이 쓰인다.
- **세트 단위로만 켜진다.** 드럼은 **3개 전부**, 목소리는 **1~6 전부** 있어야 그 음색이 활성화된다. 하나라도 없으면 그 음색 버튼은 "준비 중"으로 비활성(합성 클릭은 언제나 동작).
- 권장 스펙: **mono · 44.1kHz · 길이 ≤ 0.5s(목소리는 ≤ 0.7s) · 피크 -1dBFS 정규화 · 앞쪽 무음(선행 정적) 제거**. 앞 무음이 남아 있으면 그만큼 박이 뒤로 밀려 들린다.
- 목소리는 **영어 One·Two·Three·Four·Five·Six**(3언어 공통, B0 확정).
- **라이선스 = CC0**. 출처·라이선스는 확보 세션이 `_design_docs/05_update_backlog/05-1_add_new_function/` 에 별도 메모로 남길 것.
- 파일이 들어온 뒤 **코드 수정은 필요 없다.** `npm run build` 만 다시 돌리면 음색 버튼이 자동으로 켜진다.

| 3/4 | `500/1 · 250/0.6 · 250/0.6` | 킥 + 하이햇 2개(백비트 없음) ✅ |
| 4/4 8분세분 | 박 사이마다 `250/0.22` | 세분 = 하이햇 아주 여리게 ✅ |

- 게인 반영 후 실피크: 킥 −4.79 · 스네어 −4.29 · 하이햇 −9.48 · 세분 하이햇 −18.2 dBFS → 킥/스네어가 거의 동일 레벨, 하이햇은 한참 아래. 실기기 청감상 스네어가 세게 느껴지면 `engine.ts` `DRUM_GAIN.backbeat` 를 0.85→0.8 정도로 내리면 된다(코드 한 줄).
- **선택 최적화(미적용):** 파일 뒤쪽이 대부분 무음이다(킥 실질 277ms / 스네어 161ms / 하이햇 84ms). 트림 + `.m4a`(AAC) 인코딩 시 110KB → 10KB 안팎까지 줄일 수 있다. 다만 현재도 총 110KB(레슨 페이지 JS 42KB와 같은 수준)라 급하지 않고, WAV 는 인코더 프라이밍 패딩이 없어 타이밍이 가장 정확하다. 나중에 배킹 음원 인코딩할 때 같이 처리해도 된다.

### B4. 결정 완료 (2026-07-24)

- 목소리 겹침 → **(c) 특정 BPM↑ 자동 비활성**(임계값 튜닝, 잠정 ~180).

### 확정된 규약·상수

- **라우트:** 전용 페이지 = `/tools/{toolId}/`(ko 무접두) · `/en/tools/…` · `/ja/tools/…`. 헬퍼 `lib/urls.ts` `toolPath()`. sitemap.xml 에 자동 포함.

- **BPM 30~300**(`lib/storage.ts` `BPM_MIN`/`BPM_MAX`), 기본 90 · 4/4 · 세분 없음 · 클릭 · 볼륨 0.8.
- **목소리 자동 비활성 임계 = 180 BPM** — `engine.ts` `VOICE_MAX_BPM` 상수(여기만 고치면 됨).
- **6/8 의 BPM 기준 = 8분음표(=시각 동그라미 1개).** 강박 1, 부강박 4번째 8분.
- **세분은 소리만**(8분/셋잇단/16분) — 시각 이펙트 없음. 시각은 박 동그라미 + 아래 점 점등 + 강박 파동뿐.
- **백그라운드 탭 보정:** 예약 창을 0.1s → 1.5s 로 넓혀(`SCHEDULE_AHEAD_HIDDEN_S`) 타이머 throttle 에도 소리가 끊기지 않게 했다. 정지 시에는 아직 울리지 않은 예약 노드를 전부 취소해 꼬리 소리를 남기지 않는다.
- **샘플 프리페치(첫 박 폴백 방지, 2026-07-24 드럼 검증 중 발견·수정):** 예전엔 `start()` 안에서 샘플을 받기 시작해, 드럼을 골라도 **첫 한두 박이 합성 클릭("틱")으로 새어 나왔다.** 지금은 ① 패널 mount·음색 변경 시 `engine.prefetch()` 가 **AudioContext 없이 바이트만** 미리 받아 두고(자동재생 정책 경고 없음), ② `start()` 는 재생 상태를 먼저 UI 에 반영한 뒤 필요한 버퍼가 준비될 때까지 기다렸다가 스케줄을 시작한다(기다리는 사이 정지하면 취소). 계측상 첫 박부터 폴백 0.

### 서비스워커 메모

`public/sw.js` 의 앱셸 precache(`SHELL` 배열)에는 **메트로놈 샘플을 넣지 않았다** — `cache.addAll()` 은 목록 중 하나라도 404 면 install 전체가 실패하므로, 아직 없는 파일을 미리 적어 두면 SW 가 깨진다. 샘플은 **동일 출처 정적 자산 규칙(StaleWhileRevalidate)** 으로 첫 재생 뒤 자동 캐시된다. 파일 확보 후 오프라인 첫 사용까지 보장하고 싶다면 그때 `SHELL` 에 추가하고 `CACHE_VERSION` 을 올릴 것.
