# 베이스 세트 프롬프트 — 후속 배포 (미착수)

> 상위: [`../README.md`](../README.md) · 설계 SSOT: [`../../21_backing.md`](../../21_backing.md)

**기타 세트 배포 후 착수한다.** 지금은 규약만 남겨 둔다.

## 확정된 것

- **제외 파트: 베이스 제외** — 드럼 + 코드/기타 컴핑만 남기고 사용자가 베이스를 얹는다(21_backing R4).
- **드럼만 5트랙은 기타 세트와 공유** — `guitar/drums.md`를 그대로 쓰며 **재생성하지 않는다**.
- 파일명 규약은 기타와 동일: `{장르슬러그}_{조}_{bpm}.m4a` (매니페스트에서 `instrument:"bass"`로 구분).
- 규모 목표: 약 36트랙 / ~18 MB (기타 24 MB와 합쳐도 예산 80 MB 내).

## 미확정 (기타 배포 후 논의 — 21_backing 「남은 확인 사항」)

- **조 구성.** 베이스 커리큘럼 중심으로 좁힐지 여부: `shuffle_bounce_bass_2months`(E) · `slap_funk_bass_2months`(Em) · `beginner_bass_1month`(E) · `walking_bass_2months`(F) + 공통 Am.
- 장르별 트랙 수를 기타와 동일하게 갈지, E/Em·F 중심으로 압축할지.

## 착수 시 작업

1. 위 조 구성 확정 → 매트릭스 표 작성.
2. 기타 md를 복제해 **베이스 파트를 제거하고 컴핑/기타를 보강**하는 방향으로 프롬프트 수정.
   - 예: `no bass, no bass guitar, no upright bass` 를 프롬프트와 Exclude Styles 양쪽에 명시.
   - 베이스가 빠지면 저역이 비므로, 컴핑 악기(로즈/기타)가 **중저역을 채우지 않도록** 지시(사용자 베이스 자리 확보).
3. `../README.md` 인덱스에 베이스 표 추가 + 체크리스트 갱신.
