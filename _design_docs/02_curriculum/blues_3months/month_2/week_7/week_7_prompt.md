# 산문 강조 규칙 (기타 = 볼드 + 하이라이트)
- **★볼드 필수:** 산문(①②③)에서 핵심 용어·음이름·수치·지시를 `**볼드**`로. ①이론 풍부하게.

# 시각자료 인라인 배치 + 지판 활용
- JSON을 설명 산문 바로 아래 인라인. 각 악보 밑 `▶ 연주 안내 한 줄` 필수.
- **★지판 활용:** 손 위치는 `scale_shape` 병기 가능. 시간순 프레이즈는 tab.

# ①이론 지침
- 국어 500~700자, 3언어 동일 밀도, 토스 톤 4요소. 비유 예: 프레이징=말하듯(묻고-답하고-쉬고). 긁지 않기.

# 표준 JSON 스키마 · 필드 규약
- id: `"m2.w7.d{일}.슬러그"`. type: `scale_shape` | `tab`. meta: `{ title, stringCount:6, tuning:["E","A","D","G","B","E"], key?, tempoBpm?, notation? }`.
- 현 번호 6(저음E)~1(고음e). 도수 label에만. ⛔ 스키마 밖 필드명 금지. **여백/쉼=`rest:true`**(프레이징의 숨). 릭=`notation:"staff+tab"`. ⛔ `"rhythm"`. role root/chord_tone→파랑, target→초록, scale→색없음. ⛔ 빨강 없음.

- **4단 헤딩 정확:** day 파일은 `## ① 이론/설명` → `## ② 시각 자료` → `## ③ 오늘의 연습 (50분 루틴)` → `## ④ 팁 / 흔한 실수`.
