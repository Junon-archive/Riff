# 산문 강조 규칙 (기타 = 볼드 + 하이라이트 + 지판)
- **★볼드 필수(기존 기타 커리큘럼 방식):** 산문 본문(①②③)에서 **핵심 용어·개념어·음이름·수치·중요 지시 문장**을 `**볼드**`로. ①이론 특히 풍부하게(문단당 1~3개) — '글의 벽' 금지.
- **하이라이트:** 핵심 문장을 `<mark>…</mark>`(기타=기본 하이라이트색). `**<mark>…</mark>**` 겹침=핵심 중의 핵심. 문단당 1~2곳. 산문 `<mark>`(글) ↔ 악보 `role:target`(초록 음 점) 혼용 금지, 악보 음은 역할·도수로.
- **★지판(fretboard_diagram) 활용:** **코드 그립(Dm9·G13)은 `type:"fretboard_diagram"`으로 반드시 병기**해 손모양 각인. 순수 시간순(컴핑 리듬)은 tab.

# ①이론 지침
- 국어 500~700자, 3언어 동일 밀도, 토스 톤 4요소(동기·안심·비유·실전보상). 비유 예: 루트리스=베이스가 바닥을 깔아주니 손은 색만 칠한다. 긁지 않기.

# 표준 JSON 스키마 · 필드 규약
- id: `"m1.w1.d{일}.슬러그"`. type: `fretboard_diagram` | `tab`. meta: `{ title, stringCount:6, tuning:["E","A","D","G","B","E"], key?, tempoBpm?, notation?, feel? }`.
- 현 번호 6(저음E)~1(고음e). 개방 fret:0. 도수/음이름 label에만("R","b3","3","b7","9","13"…). ⛔ 스키마 밖 필드명 금지.
- **보이싱=`chord[]`**(대표음=최저음). **텐션(9·11·13)=`role:"color"`(노랑)** / 코드톤(3·5·b7)=`role:"chord_tone"`(파랑) / 근음=`role:"root"`. 컴핑은 `notation:"staff+tab"`. ⛔ `"rhythm"`. ⛔ 빨강 없음.
